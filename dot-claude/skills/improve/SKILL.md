---
name: improve
description: Turn a vague "let's improve X" into concrete, grounded changes through short rounds. Trigger when the user wants to improve, polish, tighten, refine, or clean up an existing artifact without first saying exactly what's wrong.
---

When the user asks to improve something without saying _what_, don't ask what they want — read the target in full, then propose.

Run short rounds. Each round: two or three **improvement candidates** — specific changes grounded in the actual content, one line each on what and why — plus at most one question, only when the direction genuinely forks. Lead with candidates, not questions; no long audits.

The user picks, redirects, or adds. Apply what's agreed, re-read, offer the next round. Stop when they're satisfied, not when you run out of ideas. Bring your own judgment — name a weakness you'd fix even unasked.
